You are a senior backend architect and Python engineer with 15+ years of production experience, with deep expertise in scalable async systems, clean architecture, and startup-grade product design.

Your task: **Design and generate a production-ready, scalable MVP backend system** for a Telegram bot called "LingoGenBot".

This MVP is meant to quickly test product-market fit but must be **production-grade, maintainable, and fully scalable** for future real-time web and voice extensions.

---

## STRICT Tech Stack:

* Python 3.11+
* aiogram (Telegram bot)
* FastAPI (backend API)
* PostgreSQL (relational database)
* Redis (queue + caching + session state)
* Docker (containerization)
* SQLAlchemy or SQLModel
* Pydantic (schemas/validation)

---

## Core Product Concept:

* Telegram bot that **connects users randomly to practice English** via text chat.
* Must deliver **real-time UX feedback**:

  * When user presses “Find Partner”, they see a **live progress indicator** (animated GIF/video with countdown).
  * Caption updates every 10–15 seconds: “Searching for a partner… Please wait.”
  * Timer stops when partner is found, session starts.
  * If no partner is found in X seconds, display friendly retry message.
* **User identity must remain anonymous** during chat sessions.
* All critical events and errors must be **forwarded to a dedicated Telegram monitoring channel** in human-readable format.

---

## Architecture & Development Rules:

1. **Clean separation of layers**:

   * Telegram Bot Layer → ONLY handles Telegram interactions
   * Backend Layer → FastAPI handles all business logic, database, matchmaking, and session management
2. Async-first architecture: use async/await everywhere
3. Clean architecture enforced:

   * routers (API endpoints)
   * services (business logic)
   * models (DB)
   * schemas (validation)
4. Logging & error handling:

   * Python logging with local persistence
   * Forward all warnings, errors, critical events to a Telegram monitoring channel with timestamp, user ID, session ID, traceback
5. Configuration management via `.env` or centralized config module
6. Dockerized environment with Postgres, Redis, and bot + API containers

## MVP Features:

### User System:

* Telegram-based user registration
* Store: id, username, created_at
* Optional profile info for future expansions

### Matchmaking:

* Redis-based queue
* If user requests a match:

  * Connect immediately if another user exists
  * Else enqueue
* Maintain **async session state** in Redis for live search timer updates
* Retry logic and safe dequeueing in case of errors

### Live Search Timer & UX Feedback:

* Animated GIF/video showing countdown
* Caption updates every 10–15 seconds
* Prevents idle waiting, improves engagement
* Async-safe, minimal system load

### Session System:

* DB model: id, user1_id, user2_id, topic, start_time, end_time, status
* Session timer: 5-minute sessions
* End-of-session notifications and rating requests (1–5)
* Store ratings in DB

### Topic System:

* At least 20 predefined topics
* Random topic assignment per session

### Messaging Relay:

* Forward messages between matched users
* Maintain anonymity
* Async-safe, fast delivery
